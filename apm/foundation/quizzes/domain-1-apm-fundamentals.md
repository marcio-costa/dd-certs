## Answer Key

| # | Answer | Explanation |
|---|--------|-------------|
| 1 | **B** | The trace-agent listens on 8126/TCP by default. 8125/UDP is DogStatsD; 4317 is OTLP gRPC. |
| 2 | **B** | USM is `env`, `service`, `version`. The other tag combos are useful but not USM. |
| 3 | **B** | A trace is the tree of spans for an end-to-end request. A span is the unit; a trace is the graph. |
| 4 | **C** | The trace-agent process inside the Datadog Agent receives and forwards spans. |
| 5 | **B** | Ingested spans (Live Search) live for 15 minutes before they roll off, unless retained. |
| 6 | **C** | Custom retention filters keep spans for 15 days. |
| 7 | **D** | Trace metrics (computed from 100% of ingested data) are retained for 15 months. |
| 8 | **C** | The intelligent retention filter keeps indexed spans for 30 days. |
| 9 | **B** | `DD_VERSION` is the canonical USM env var. The others don't exist. |
| 10 | **C** | Go has no JVM-style bytecode injection; you import contrib packages like `httptrace` and `chitrace`. |
| 11 | **A** | "Community tracer" usually means OpenTelemetry SDK or a third-party tracer that sends Datadog-compatible spans, not the Datadog `dd-trace-*` lib. |
| 12 | **C** | Head-based sampling decides at the start of the root span and propagates the decision via headers. |
| 13 | **A** | The default Trace Explorer view is Live Search over the last 15 minutes of ingested spans. |
| 14 | **B** | The OTel Collector path: OTel SDK → Collector with Datadog exporter → backend. |
| 15 | **B** | Priority for auto-version (Agent 7.52+): explicit `DD_VERSION` > container image tag > Git commit SHA. |
| 16 | **C** | Custom span tags use `@` (e.g., `@order.id:42`). Reserved attributes (like `service:`) have no prefix. |
| 17 | **C** | USM unifies APM, logs, processes, and infra metrics through `env`/`service`/`version` tags. |
| 18 | **B** | The Datadog Lambda Extension is the recommended way to forward traces from Lambda functions. |
| 19 | **A** | DogStatsD = 8125/UDP. Trace-agent = 8126/TCP. |
| 20 | **C** | There is no Kafka-based ingestion path for traces. The other three are valid architectures. |
